# 销售管理与报价系统 — 架构方案

> 最后更新：2026-06-22

---

## 一、角色与权限

| 角色 | 职责 | 核心权限 |
|---|---|---|
| **销售经理** | 销售管理、销售支持、组织报价与跟进 | 管理所有销售机会、创建报价单、推动销售流程、生成报价表 |
| **方案经理** | 维护组件/物料数据库、编制技术方案和概算表 | 物料CRUD、编制/编辑概算表（所有版本）、上传项目方案截图 |
| **交付经理** | 赢单后接手项目交付 | 更新交付节点进度、确认验收回传、录入实际成本（按概算表结构） |
| **总监** | 审核概算表、审核物料变更、系统管理 | 审核所有版本的概算表、审核物料CRUD、用户管理/基础配置 |

> 注：区域销售没有系统账号，其信息由销售经理统一录入。总监同时担任系统管理员角色。

## 四、项目状态流转

```
                      ┌──────────┐
                      │  过程中   │◄──── 新建/编辑中
                      └────┬─────┘
                           │
             ┌─────────────┼──────────────┐
             ▼             ▼              ▼
         ┌──────┐     ┌──────┐      ┌──────┐
         │  赢  │     │  输  │      │ 冻结 │
         └──────┘     └──┬───┘      └──┬───┘
                         │             │
                    ┌────┴────┬───────┐ │ 可恢复：
                    ▼         ▼       ▼ │ 过程中 / 赢 / 输
                 ┌──────┐ ┌──────┐ ┌──────┐
                 │ 丢单  │ │ 取消 │ │ 放弃 │
                 │(竞对) │ │(客户)│ │(我方)│
                 └──────┘ └──────┘ └──────┘
```

- **赢** / **输**（含丢单、取消、放弃）/ **冻结** 的项目 → 从销售管理模块转移到销售分析模块
- **过程中** → 留在销售管理模块，继续推进
- **冻结** → 可恢复为过程中 / 赢 / 输，恢复后回到对应模块

## 七、交付管理

### 交付节点生命周期（15个节点）

```
赢单 → 交付经理接手
  → Handover → 合同签订 → Kickoff → 方案细化 → 技术会签
  → 详细设计 → 设计评审 → 制造采购 → 组装调试 → 出厂验收
  → 包装发货 → 现场安调 → 验收整改 → 终验收 → 项目总结 → 关闭
```

| 序号 | 节点 | 说明 |
|---|---|---|
| 1 | Handover | 项目交接（销售→交付） |
| 2 | 合同签订 | 正式合同签署 |
| 3 | Kickoff | 项目启动会 |
| 4 | 方案细化 | 详细方案设计 |
| 5 | 技术会签 | 客户确认技术方案 |
| 6 | 详细设计 | 出图/编程 |
| 7 | 设计评审 | 内部评审 |
| 8 | 制造采购 | 物料采购 + 生产制造 |
| 9 | 组装调试 | 厂内组装和预调试 |
| 10 | 出厂验收 | 客户到厂预验收 |
| 11 | 包装发货 | 设备包装发运 |
| 12 | 现场安调 | 现场安装调试 |
| 13 | 验收整改 | 验收问题整改 |
| 14 | 终验收 | 客户最终验收签字 |
| 15 | 项目总结 | 项目复盘、成本核算 |

### 节点管理机制

交付经理负责维护每个节点的：
- **状态**：未开始 / 进行中 / 已完成 / 已延期（点击循环切换）
- **计划日期**：可设定和修改，每次修改自动记录变更历史
- **实际日期**：状态改为"已完成"时，系统自动填入当前日期
- **历史记录**：仅记录计划日期修改历史，节点状态变更不记历史

### 实施计划审批流

```
交付经理设定所有节点的计划时间
  → 点击"提交实施计划"
  → 状态变为"待审批"
  → 总监审批
      ├── 通过 → 所有节点计划时间锁定（可修改，但修改后系统记录变更历史）
      └── 驳回 → 退回交付经理修改后重新提交
```

- 审批通过后，交付经理如需修改计划时间，可以修改，系统记录修改历史
- 计划锁定后，日期显示为只读文本

### 成本对比审批流

成本对比基于**报价表的逐条项次**（GroupItem），不是按节点分配成本。

```
交付经理在各报价项次中输入实际成本
  → 点击"提交成本对比"
  → 状态变为"待审批"
  → 总监审批
      ├── 通过 → 成本数据锁定（只读），作为交付经理和方案经理的考核依据
      └── 驳回 → 退回修改后重新提交
```

### 实际成本对比规则

| 对比维度 | 差异红线 | 说明 |
|---|---|---|
| **分项成本** | **-5% ~ +10%** | 超出此范围标红显示"超标" |
| **总成本** | **-2.5% ~ +5%** | 超出此范围标红显示"超出正常范围" |

- 只比较**成本**，不比较售价
- 偏差项次标红，正常显示绿色
- 仅在成本对比审批通过后数据锁定，作为考核依据

### 项目完成条件

同时满足以下两个条件时，可点击「完成项目」按钮将项目标记为已完成：
1. **节点15（项目总结）状态为"已完成"**
2. **成本对比审批通过**

已完成的项目自动从交付管理首页的"进行中"列表移至"已完成"列表。

### 交付管理页面结构

- **首页**（`/delivery`）：列表化展示，分为"进行中"和"已完成"两个标签
  - 每个项目卡片显示：进度圆环、客户名、项目名、合同金额、计划/成本审批状态
  - 点击卡片进入详情页
- **详情页**（`/delivery/:id`）：两个 Segmented 标签
  - **实施计划**：15节点时间轴 + 审批状态条
  - **成本对比**：报价表项次逐条对比 + 审批状态条

## 八、审核工作流

### 概算表审核（所有版本都需要）

**版本生命周期：**
```
方案经理编制（草稿状态，可见但不可生成报价单）
        │
        ▼ 手动点击"提交审核"
方案经理 提交审核 ──► 总监 审核
                          ├── 通过 → 版本生效，可生成报价表
                          └── 驳回 → 退回方案经理修改后重新提交
```

**特殊规则：**
- 每次点击"提交审核"生成一个新版本号，中间修改保存不递增版本号
- 方案经理的报价编制页面**不显示生成报价表按钮**
- 销售经理可以查看所有版本的概算表，但总监未审批或审批不通过的版本**不可生成报价表**
- **毛利率 GP3 < 15%** 时，必须总监审批通过才能生成报价表
- 未审核通过或草稿状态的版本：**可见但不可生成报价表**

### 物料/组件数据库审核
```
方案经理 提交物料变更 ──► 总监 审核
                                    ├── 通过 → 物料生效
                                    └── 驳回 → 退回修改
```

### 交付管理两级审批
1. **实施计划审批**：交付经理提交节点计划时间 → 总监审批 → 通过后计划时间锁定
2. **成本对比审批**：交付经理提交报价项次实际成本 → 总监审批 → 通过后成本数据锁定，作为考核依据

## 九、数据分析模块

### 9.1 赢单分析
- 赢单金额/数量趋势（月度/季度）
- 赢单平均毛利率（GP3）
- 按客户行业/产品线分布

### 9.2 输单分析
三大原因分类：

| 输单类型 | 说明 | 分析维度 |
|---|---|---|
| **丢单** | 输给竞争对手 | 竞对名称、丢单阶段、价格差距原因 |
| **取消** | 客户取消项目 | 取消阶段、取消原因（预算/需求变更/其他） |
| **放弃** | 我方主动退出 | 放弃原因（条件无法达成/风险评估过高/资源不足） |

- 输单率趋势
- 按输单原因分布
- 丢单竞争对手分析
- 输单阶段分布（哪个阶段最容易输）

### 9.3 冻结分析
- 冻结项目数量/金额
- 冻结时长分布
- 冻结原因

### 9.4 销售漏斗分析
- 各阶段机会数量/金额
- 阶段转换率（信息→线索→机会→投标→议价→中标→赢）
- 平均停留时长（每个阶段）

### 9.5 销售预测
- 基于当前漏斗的加权预测
- 按销售经理/区域汇总

## 十、推荐开发路线

```
Phase 1 ─ 报价编制页面升级
├── 项目方案上传（布置图 PDF/PNG，嵌入预计定标列后）
├── 新增"保存"按钮（不递增版本号）
├── 新增"提交审核"按钮（递增版本号、状态变为待审批）
├── 新增"新报价"按钮（创建空白报价单）
├── 审核状态标签显示（草稿/待审批/已通过/已驳回）
├── 版本管理（提交审核才生成版本号，草稿不可导出）
├── 按钮权限：方案经理可见保存+提交审核，销售经理可见保存+生成报价表
├── GP3<15% 特殊审批规则
├── 取消"上传文件"按钮
└── 新增项目名称字段、更新阶段定义为 信息→线索→机会→投标→议价→中标

Phase 2 ─ 销售机会管理 + 报价列表
├── 销售机会列表页（含全部字段）
├── 每行"编制报价表"按钮 → 创建/打开对应项目的报价
├── 报价列表页（报价菜单入口）
│   ├── 展示所有报价：客户/项目/版本/状态/总金额/更新时间
│   ├── 按状态筛选（草稿/待审批/已通过/已驳回）
│   └── 点击进入报价编制页面（进行中可编辑，已完结只读）
├── 机会 CRUD + 阶段推进
├── 销售漏斗看板
└── 机会→报价联动逻辑：
    ├── 无报价 → 创建新报价
    └── 有报价 → 打开最新版本

Phase 3 ─ 审核工作流
├── 概算表审核（方案经理→总监）
├── 物料审核（→总监）
├── 待审核列表
└── 审核历史

Phase 4 ─ 交付管理
├── 项目交接（赢单后交付经理接手）
├── 交付节点跟踪：
│   设计 → 内部评审 → 客户会签 → 采购生产 → 装配预调
│   → 预验收 → 发货 → 现场安调 → 验收整改 → 验收 → 项目总结
├── 验收回传
├── 实际成本录入与对比（按概算表结构，分项 -5~+10 / 总成本 -2.5~+5 红线）
└── 附件管理（赢单项目下挂 客户需求书/技术方案/技术协议/商务合同）

Phase 5 ─ 客户管理
├── 客户档案 + 联系人
├── 客户分级
└── 客户历史

Phase 6 ─ 物料数据库管理
├── 物料 CRUD
├── 编码规则
├── 版本管理
└── 变更审核

Phase 7 ─ 销售分析
├── 赢/输/冻结分析
├── 漏斗分析
├── 预测
└── 数据看板

Phase 8 ─ 系统管理
├── 用户/角色/权限
├── 基础配置
└── 操作日志
```
